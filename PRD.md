# 学生积分管理系统 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品名称
学生积分管理系统

### 1.2 产品定义
一个用于管理和展示学生名称及其对应积分的系统，包含后端数据管理和前端展示功能。

### 1.3 目标用户
- 教师/管理员：用于管理学生积分数据
- 学生/家长：查看学生积分信息

---

## 2. 功能需求

### 2.1 后端需求

#### 2.1.1 学生管理
- **添加学生**：支持添加新学生，记录学生名称和初始积分
- **编辑学生**：修改学生信息（名称、积分）
- **删除学生**：删除指定学生记录
- **查询学生**：
  - 获取所有学生列表
  - 按名称搜索学生
  - 按积分排序（升序/降序）

#### 2.1.2 积分管理
- **更新积分**：修改学生的积分值
- **增加/减少积分**：支持积分的增减操作
- **积分历史**（可选）：记录积分变动日志

#### 2.1.3 数据接口
后端需提供以下 API 接口：
```
GET    /api/students           - 获取所有学生
GET    /api/students?name=xxx  - 按名称搜索
GET    /api/students/:id       - 获取单个学生详情
POST   /api/students           - 创建新学生
PUT    /api/students/:id       - 更新学生信息
DELETE /api/students/:id       - 删除学生
```

### 2.2 前端需求

#### 2.2.1 页面功能
- **学生列表页**：
  - 表格显示所有学生的名称和积分
  - 搜索功能：按学生名称搜索
  - 排序功能：按积分升序/降序排列
  - 分页展示（可选）

#### 2.2.2 交互功能
- **添加学生**：弹窗或表单添加新学生
- **编辑学生**：点击编辑按钮修改学生信息
- **删除学生**：确认后删除学生记录
- **积分调整**：快速增加/减少学生积分

#### 2.2.3 UI 设计
- 简洁清晰的表格布局
- 响应式设计（支持移动设备）
- 操作按钮明确（编辑、删除、保存）
- 确认弹窗防止误操作

---

## 3. 技术栈建议

### 后端
- **框架**：Node.js (Express/Koa) 或 Python (Flask/Django)
- **数据库**：SQLite / MongoDB / PostgreSQL
- **语言**：JavaScript/TypeScript 或 Python

### 前端
- **框架**：React / Vue / Vue 3
- **样式**：CSS / TailwindCSS / Ant Design / Element UI
- **状态管理**：Context API / Pinia / Vuex（可选）
- **HTTP 客户端**：Axios / Fetch API

---

## 5. 非功能需求

### 5.1 性能
- 列表加载时间 < 1 秒
- 支持至少 1000 条学生记录

### 5.2 可用性
- 操作需要确认提示（删除等危险操作）
- 错误消息清晰明确
- 操作成功有反馈提示

### 5.3 安全性
- 数据验证（前后端都需要）
- 防止 XSS 攻击
- 敏感数据不在前端暴露

---

## 6. 成功标准

- [ ] 后端能正确增删改查学生数据
- [ ] 前端能正确显示学生列表
- [ ] 前端能实现搜索和排序功能
- [ ] 用户能通过前端增删改学生积分
- [ ] 系统稳定无明显 Bug
